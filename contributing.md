# Contributing to the Academic MCP Guide and Directory

## How to Suggest a New Server 

We accept changes through the [GitHub flow model](https://docs.github.com/en/get-started/using-github/github-flow). 

Open an issue, submit a pull request, or email loisw@uchicago.edu with the following information:

- **Server name**
- **Link** to the server's GitHub repo or documentation 
- **Data source** (what content or system it connects to)
- **Developer** (publisher, institution, or community-maintained)
- **Access** (free, API key required, institutional subscription, etc.)

## Criteria for Inclusion

To be listed in the directory, a server should:

- Be relevant to academic libraries, research, or higher education
- Be a functioning, connectable MCP server 
- Have a public repository or documentation link

## Flagging Outdated or Broken Entries

If a listed server is no longer maintained or its link is broken, please open an issue so we can update or remove it.

## Contributing to the Guide

This resource is more than a server directory. It's meant to be a living guide 
for the academic community. Contributions to the guide content itself are 
welcome, including sections on responsible use, getting started tutorials, 
institutional use cases, or anything else relevant to MCP in academic contexts.

To suggest additions or edits to the guide, open an issue describing what you'd 
like to add or change, submit a pull request with your proposed content, or email loisw@uchicago.edu.
